# 🎲 [Vibe Coding Prompt] 음악의 마블 (Music Marble) 보드게임 웹앱 개발

당신은 세계 최고의 프론트엔드 개발자이자 인터랙티브 웹 애니메이션 전문가, 그리고 UI/UX 디자이너입니다. 
지금부터 음악 이론 기호를 보드게임을 통해 자연스럽게 학습할 수 있는 **'음악의 마블 (Music Marble)'** 웹앱을 Vibe Coding 방식으로 개발할 것입니다. 
아래의 기획서와 단계별 지침을 정확히 숙지하고, 완벽하게 동작하는 코드를 작성해주세요.

---

## 1. 🛠️ 기술 스택 및 디자인 시스템 (Design System)
* **프론트엔드**: HTML5, CSS3, Vanilla JavaScript (또는 React - 개발 환경에 맞게 선택)
* **디자인 스타일**: 노션(Notion)처럼 깔끔하고 미니멀한 모바일 친화적 UI/UX를 베이스로 하되, 보드판과 게임 요소는 화려하고 직관적으로 디자인합니다.
* **레이아웃**: 화면 중앙에 보드판이 위치하며, 하단에 컨트롤(주사위 등)이 있는 구조.
* **에셋**: 보드말은 '귀여운 음표' 또는 '악기' 형태의 SVG로 구현합니다. 음악 기호들은 가독성 높은 폰트나 SVG 아이콘으로 타일 위에 명확히 표시되어야 합니다.

---

## 2. 🎮 핵심 게임 로직 및 조작 (Core Mechanics)
1.  **이동 방식**: 사용자가 **`우측 방향키(→)`**를 누를 때마다 보드말이 앞으로 **한 칸씩** 이동합니다. 
2.  **보드말 애니메이션**: 한 칸 이동할 때마다 보드말(SVG)이 통통 튀어 오르는(Bouncing) 귀여운 CSS/SVG 애니메이션이 적용되어야 합니다.
3.  **주사위 인터랙션**: 별도의 '주사위 굴리기' 버튼이나 화면 터치 시, 3D 주사위가 굴러가는 화려한 애니메이션 팝업이 나타나고 결과값을 보여줍니다. (방향키 이동과 연계하여 튜토리얼 또는 보조 수단으로 활용)

---

## 3. 🎵 음악 기호 타일별 특수 이벤트 및 화려한 CSS 인터랙션
보드말이 특정 음악 기호가 있는 칸에 도착하면, 각 기호의 의미(음악 이론)에 완벽히 부합하는 기능이 실행되어야 하며, **반드시 매우 화려하고 몰입감 있는 CSS 애니메이션 인터랙션**이 동반되어야 합니다.

| 기호명 | 타일 도착 시 로직 (기능) | 매칭되는 화려한 CSS 애니메이션 인터랙션 요구사항 |
| :--- | :--- | :--- |
| **다중쉼표 4** | 현재 위치에서 **4칸 앞으로** 자동 전진 | 캐릭터가 로켓을 타고 부스터를 뿜으며 타일 4개를 빠르게 날아가는 이펙트 |
| **끝 도돌이표** | 보드판 내의 **'시작(Pre) 도돌이표'**가 있는 칸으로 강제 이동 | 화면에 테이프 되감기(Rewind) 잔상 효과가 발생하며 캐릭터가 뒤로 슉! 빨려 들어감 |
| **달세뇨 (D.S.)** | **'세뇨(Segno)'**가 있는 칸으로 즉시 이동 | 캐릭터가 공중으로 떠올라 화려한 'S'자 궤적을 그리며 타겟 칸으로 날아감 |
| **세뇨** | (달세뇨의 목적지 역할 - 자체 이벤트 없음) | 도착 시 깃발이 꽂히며 반짝이는 파티클 효과 |
| **코다 (Coda)** | **'두 번째 코다'**가 있는 칸으로 순간 이동 | 타일 바닥에 우주 포털(블랙홀)이 열리고 캐릭터가 빠진 뒤, 목적지 포털에서 튀어나옴 |
| **다카포 (D.C.)** | 보드의 맨 **첫 번째 칸(Start)**으로 돌아감 | 화면 전체가 잠시 암전(Flash)되었다가, 시작점에 화려한 스포트라이트가 비추며 캐릭터가 등장 |
| **제 1괄호** | **'제 2괄호'**가 있는 칸으로 건너뜀 | 캐릭터가 스프링을 밟은 듯, 화면 상단을 뚫을 듯이 크고 높은 포물선을 그리며 슈퍼 점프 |
| **제 2괄호** | (제 1괄호의 목적지 역할) | 도착 시 "Success!" 텍스트와 함께 폭죽 이펙트 터짐 |
| **페르마타** | 다음 **2번의 턴 동안 이동 불가** (쉬기) | 캐릭터가 얼음 큐브에 갇히고, 머리 위에 'Zzz' 수면 마크가 떠오르는 애니메이션 |
| **더블세뇨** | **'두 번째 더블세뇨'**가 있는 칸으로 이동 | 두 개의 빛줄기가 캐릭터를 감싸며 홀로그램처럼 사라졌다가 목적지에 재구성됨 |
| **더블코다** | **'두 번째 더블코다'**가 있는 칸으로 이동 | 화면에 거대한 웜홀이 생성되어 캐릭터를 빨아들이는 시네마틱 이펙트 |

## 4. 🚀 단계별 구현 지침 (Step-by-Step Implementation)
프롬프트를 입력받은 AI는 다음 순서대로 코드를 생성하고 구현을 진행하세요.

### **Phase 1: 마크업 및 보드판 레이아웃 렌더링**
* 음악 기호들이 배치된 20~30칸 규모의 보드판 배열(Array)을 설계합니다.
* Grid 또는 Flexbox를 사용하여 맵(뱀파이어 서바이버 맵이나 브루마블 맵 형태)을 화면에 렌더링합니다.
* 노션(Notion) 스타일의 부드러운 그림자와 둥근 모서리(border-radius)를 적용하여 타일을 디자인합니다.

### **Phase 2: 캐릭터 렌더링 및 키보드 조작 로직**
* 보드말을 SVG 렌더링하고, 현재 인덱스(State)에 따라 보드판 위의 좌표(Top, Left)로 부드럽게 이동(Transition)하도록 만듭니다.
* `keydown` 이벤트를 리스닝하여 `ArrowRight` 입력 시, 보드말이 통통 튀며(transform: translateY) 1칸씩 전진하는 기본 로직을 작성합니다.
* 주사위 UI 요소를 구현하고 굴리기 애니메이션을 적용합니다.

### **Phase 3: 게임 룰 및 이벤트 리스너(음악 기호) 연동**
* 캐릭터의 이동 애니메이션이 끝나는 시점(`transitionend` 또는 `setTimeout`)에 현재 타일의 '음악 기호'를 판별하는 함수를 실행합니다.
* 기호에 맞는 타겟 인덱스로 상태(State)를 업데이트하고 강제 이동시키는 로직을 작성합니다. (예: 다카포 도착 -> state = 0)

### **Phase 4: 시각적 극대화 및 폴리싱 (CSS Animations)**
* 이 기획의 핵심은 **'화려한 인터랙션'**입니다. Phase 3의 이벤트가 발생할 때, 단순 위치 이동이 아니라 각 기호에 명시된 시각적 효과(CSS Keyframes, Particle 렌더링 등) 클래스가 동적으로 `body`나 `캐릭터 요소`에 부여되도록 코드를 정교하게 다듬으세요.
