# 반복기호의 마블 UI 리마스터 바이브코딩 프롬프트

## 사용 목적

이 프롬프트는 현재 React + TypeScript + Vite 기반 웹앱인 `반복기호의 마블`의 전체 UI를 밝고 화려한 한국형 캐주얼 보드게임 감성으로 리마스터하기 위한 작업 지시서다. 참고 방향은 `모두의 마블`처럼 즉각적으로 게임판이 눈에 들어오고, 타일과 플레이어 말, 주사위, 현재 턴 패널이 생동감 있게 보이는 모바일 보드게임 스타일이다.

단, 특정 게임의 로고, 캐릭터, 일러스트, UI 배치, 아이콘, 맵, 색상 조합을 그대로 복제하지 않는다. 결과물은 `음악의 마블 / 반복기호 학습 보드게임`만의 독자적인 음악 교육 게임 UI여야 한다.

## 최종 목표

현재 웹앱을 다음 느낌으로 리마스터한다.

- 게임판이 첫 화면의 주인공처럼 가장 크게 보인다.
- 각 칸은 평평한 학습 카드가 아니라, 모바일 보드게임의 부동산 타일처럼 광택, 테두리, 색상 띠, 상승감이 있다.
- 오른쪽 조작 패널은 좁고 강한 HUD처럼 보이며, 현재 턴, 주사위, 남은 이동 수, 이벤트 로그가 즉시 읽힌다.
- 주사위와 플레이어 말은 장난감 같은 입체감과 움직임을 가진다.
- 음악기호는 교육용 정보로 선명하게 유지하되, 게임 보상/이벤트처럼 즐겁게 보인다.
- 데스크톱, 태블릿, 모바일에서 모두 깨지지 않고, 텍스트 겹침과 넘침이 없어야 한다.

## 작업 방식

너는 시니어 프론트엔드 엔지니어이자 게임 UI 디자이너다. 사용자에게 추가 질문하지 말고, 아래 지시를 기준으로 스스로 판단해서 구현한다. 기존 게임 로직은 보존하고, UI와 애니메이션, 레이아웃 중심으로 개선한다.

작업 중 다음 원칙을 지킨다.

- 게임 규칙, 이동 로직, 주사위 결과, 보드 칸 데이터는 변경하지 않는다.
- UI 리마스터에 필요한 컴포넌트 구조 변경은 허용하되, 불필요한 대규모 리팩터링은 하지 않는다.
- 새 패키지는 꼭 필요할 때만 추가한다. 기존 `lucide-react`와 CSS를 우선 사용한다.
- 원작 게임의 로고, 캐릭터, 고유 아이콘, 화면 구성을 복제하지 않는다.
- "밝고 화려한 캐주얼 보드게임풍"으로 재해석한다.
- 한 가지 색상만 반복하는 단조로운 팔레트를 피한다.
- 모든 버튼과 패널은 실제 플레이 중 반복 사용하기 쉬운 크기와 위치로 만든다.

## 디자인 컨셉

### 키워드

- 프리미엄 모바일 보드게임
- 밝은 도시형 게임판
- 음악 테마 놀이공원
- 반짝이는 보상감
- 주사위와 말의 장난감 같은 입체감
- 수업용 학습 앱이지만 게임처럼 몰입되는 화면

### 금지할 것

- 특정 상용 게임의 로고, 캐릭터, 맵, 고유 UI를 그대로 따라 하지 말 것
- 과도한 어두운 배경, 단조로운 파란색 일색 UI 금지
- 큰 마케팅 히어로 섹션 금지
- 게임 설명 텍스트가 화면 대부분을 차지하는 구성 금지
- 카드 안에 또 다른 카드가 들어가는 과한 중첩 금지
- 텍스트가 버튼, 칸, 패널 밖으로 삐져나오거나 겹치는 상태 금지

## 시각 시스템

### 컬러 토큰

`src/styles/globals.css`의 `:root`에 다음 방향의 CSS 변수를 추가하거나 기존 변수를 리마스터한다.

```css
:root {
  --sky-top: #76d7ff;
  --sky-mid: #b9efff;
  --stage-cream: #fff7d8;
  --tile-white: #ffffff;
  --tile-blue: #3d8bff;
  --tile-red: #ff4f78;
  --tile-green: #34c978;
  --tile-gold: #ffd45a;
  --tile-purple: #8b6cff;
  --ink: #14213d;
  --muted-ink: #667085;
  --panel: rgba(255, 255, 255, 0.9);
  --panel-border: rgba(20, 33, 61, 0.14);
  --glow-blue: rgba(61, 139, 255, 0.32);
  --glow-gold: rgba(255, 212, 90, 0.45);
  --shadow-pop: 0 14px 30px rgba(20, 33, 61, 0.18);
  --shadow-hard: 0 6px 0 rgba(20, 33, 61, 0.16);
}
```

색상은 이 값을 그대로 써도 되지만, 화면 전체가 한 가지 색으로 보이면 반드시 조정한다. 보드판은 하늘색, 골드, 화이트, 코랄, 그린, 퍼플이 균형 있게 섞여야 한다.

### 배경

앱 전체 배경은 게임 월드의 하늘과 무대처럼 구성한다.

- 상단은 밝은 하늘색
- 중앙은 보드판을 받치는 따뜻한 크림색
- 하단은 은은한 골드 톤
- 배경 장식은 CSS radial-gradient, linear-gradient, 작은 별빛/음표 패턴 정도로만 만든다.
- 커다란 보라색 오브, 흐릿한 추상 블롭, 과도한 다크 그라데이션은 쓰지 않는다.

예시 방향:

```css
body {
  background:
    radial-gradient(circle at 18% 12%, rgba(255, 255, 255, 0.9), transparent 22rem),
    radial-gradient(circle at 82% 8%, rgba(255, 212, 90, 0.38), transparent 18rem),
    linear-gradient(180deg, var(--sky-top) 0%, var(--sky-mid) 36%, var(--stage-cream) 100%);
}
```

### 타이포그래피

- 한국어 가독성이 우선이다.
- 외부 폰트 추가 없이 시스템 폰트를 우선 사용한다.
- 제목은 굵고 둥근 인상으로, 본문과 패널 텍스트는 짧고 선명하게 유지한다.
- 칸 안의 텍스트는 한글명이 작아도 읽혀야 하며, 넘치면 글자 크기와 줄바꿈을 조정한다.

## 레이아웃 리마스터 단계

### 1단계: 전체 앱 프레임 변경

대상 파일:

- `src/App.tsx`
- `src/styles/globals.css`

작업:

1. `.app-shell`을 게임 스테이지처럼 만든다.
2. 데스크톱에서는 보드가 화면 왼쪽 대부분을 차지하고, 오른쪽 HUD 패널은 280px 안팎으로 유지한다.
3. 메인 레이아웃은 다음 비율을 목표로 한다.
   - 보드 영역: `minmax(720px, 1fr)`
   - HUD 영역: `280px`에서 `320px` 사이
4. 세로 높이는 가능한 한 브라우저 높이를 적극 활용한다.
5. 보드와 HUD 사이 간격은 12px에서 18px 사이로 유지한다.
6. 상단 설명형 히어로 느낌을 줄이고, 실제 게임판과 조작 패널이 첫 화면에서 바로 보이게 한다.

완료 기준:

- 데스크톱 첫 화면에서 보드판이 가장 큰 시각 요소다.
- 오른쪽 패널은 좁지만 버튼과 주사위는 충분히 크다.
- 중앙 안내 텍스트는 보드 안쪽 빈 공간에 자연스럽게 들어간다.

### 2단계: 보드판을 게임 월드처럼 리디자인

대상 파일:

- `src/components/Board.tsx`
- `src/styles/board.css`

작업:

1. 보드판 바깥 컨테이너에 두꺼운 골드/화이트 프레임을 준다.
2. 보드판 안쪽은 연한 크림색 또는 밝은 하늘색의 게임판 바닥으로 만든다.
3. 현재 40칸 경로 구조는 유지한다.
4. 보드 외곽에는 아주 은은한 입체 그림자를 추가한다.
5. 보드 중앙에는 `MUSIC MARBLE`, `반복기호의 마블` 텍스트를 유지하되, 모바일 게임 로비가 아니라 보드 중앙 로고처럼 작고 선명하게 배치한다.
6. 보드 중앙의 보조 배지는 `자동 이동`, `주사위`, `기호 효과`처럼 유지하되, 작은 캡슐 배지로 만든다.

스타일 방향:

- 보드 프레임은 흰색과 골드의 이중 테두리
- 바깥 그림자는 부드럽고 명확하게
- 중앙은 너무 비어 보이지 않도록 희미한 오선지 패턴이나 음표 워터마크를 CSS로 추가

완료 기준:

- 보드가 단순한 CSS grid가 아니라 실제 보드게임판처럼 보인다.
- 중앙 안내 영역은 게임 플레이를 방해하지 않는다.

### 3단계: 개별 칸을 부동산 타일처럼 리마스터

대상 파일:

- `src/components/BoardCell.tsx`
- `src/styles/board.css`

작업:

1. 모든 칸은 둥근 사각형 타일로 유지하되, 상단에 색상 띠를 넣는다.
2. 일반 칸은 비어 있더라도 완전히 밋밋하지 않게 연한 광택과 작은 패턴을 넣는다.
3. 음악기호 칸은 기호 종류에 따라 색상군을 다르게 한다.
   - 반복기호 계열: 파랑
   - 이동기호 계열: 보라
   - 코다/세뇨 계열: 골드 또는 레드
   - 쉼표/옥타브 계열: 그린
   - 특수 종료/시작 칸: 골드와 코랄
4. `Start`와 `Fine` 칸은 특별 타일처럼 강조한다.
5. 현재 플레이어가 있는 칸은 빛나는 링, 바닥 그림자, 살짝 튀는 애니메이션을 준다.
6. 음악기호 이미지와 텍스트가 겹치지 않게 한다.
7. 한글명은 기호 아래 작고 선명하게 표시한다.

구현 힌트:

- `BoardCell`에서 `cell.type`, `cell.symbolId`, `cell.kind`에 따라 클래스명을 세분화한다.
- CSS 클래스 예시:

```tsx
<article className={`board-cell cell-${cell.kind} symbol-${cell.symbolId ?? 'none'} ${isActive ? 'is-active' : ''}`}>
```

완료 기준:

- 모든 칸이 게임 타일처럼 보인다.
- 음악기호 칸은 한눈에 특별 칸으로 구분된다.
- 일반 칸에는 불필요한 숫자나 라벨이 다시 생기지 않는다.

### 4단계: 플레이어 말을 장난감 피스처럼 강화

대상 파일:

- `src/components/PlayerToken.tsx`
- `src/styles/board.css`
- `src/styles/animations.css`

작업:

1. 플레이어 말은 납작한 점이 아니라 입체 피스처럼 보이게 한다.
2. 각 플레이어 색상은 보석처럼 선명하게 만든다.
3. 말 아래에 작은 타원 그림자를 추가한다.
4. 현재 턴 플레이어 말에는 은은한 펄스 링을 준다.
5. 이동 중에는 말이 칸 사이를 살짝 점프하는 느낌을 유지하거나 강화한다.

완료 기준:

- 말이 어느 칸에 있는지 즉시 보인다.
- 여러 명이 같은 칸에 있어도 겹침이 심하지 않다.

### 5단계: 오른쪽 HUD 패널 리마스터

대상 파일:

- `src/components/PlayerStatusBar.tsx`
- `src/components/DicePanel.tsx`
- `src/components/EventLog.tsx`
- `src/styles/globals.css`

작업:

1. 오른쪽 패널은 하나의 게임 HUD처럼 보이게 한다.
2. 각 섹션은 분리되어 보이되 과한 카드 중첩은 피한다.
3. 현재 턴 영역은 플레이어 색상과 이름이 강하게 보이게 한다.
4. 주사위 영역은 중앙에 3D 주사위를 놓고, 아래에 남은 이동 수와 버튼을 둔다.
5. `주사위 굴리기` 버튼은 가장 강한 CTA로 만든다.
6. 리셋과 소리 버튼은 보조 버튼으로 작고 명확하게 둔다.
7. 이벤트 로그는 어두운 콘솔이 아니라 밝은 알림 피드처럼 만든다.

버튼 스타일 방향:

- 주 버튼: 파란색 또는 코랄색 그라데이션, 하단 눌림 그림자
- 보조 버튼: 흰색, 얇은 테두리, 아이콘 중심
- disabled 상태: 흐릿하지만 버튼 크기는 유지

완료 기준:

- 플레이어는 오른쪽 패널만 보고도 현재 턴, 주사위 상태, 남은 이동 수를 이해할 수 있다.
- 패널 너비는 너무 넓어지지 않는다.

### 6단계: 3D 주사위 연출 개선

대상 파일:

- `src/components/DicePanel.tsx`
- `src/styles/globals.css`
- `src/styles/animations.css`

작업:

1. 현재 구현된 3D 큐브 주사위를 유지한다.
2. 굴리는 중에는 큐브가 위로 살짝 튀고, X/Y/Z축으로 빠르게 회전한다.
3. 주사위가 멈출 때는 결과 눈이 정면으로 자연스럽게 안착해야 한다.
4. 주사위 주변에 너무 과하지 않은 빛 번짐을 추가한다.
5. 주사위 점은 선명한 남색 또는 검정 계열로 유지한다.

완료 기준:

- 숫자 박스처럼 보이지 않고 실제 정육면체 주사위처럼 보인다.
- 굴림 애니메이션 중에도 레이아웃이 흔들리지 않는다.

### 7단계: 이벤트와 학습 카드 연출 리마스터

대상 파일:

- `src/components/EventOverlay.tsx`
- `src/components/LearningCard.tsx`
- `src/styles/globals.css`
- `src/styles/animations.css`

작업:

1. 음악기호 이벤트가 발생하면 작은 보상 팝업처럼 보여준다.
2. 팝업에는 기호 이미지, 기호명, 한글명, 짧은 효과 설명을 넣는다.
3. 배경은 반투명 딤 처리하되, 게임 흐름이 끊기지 않게 빠르게 등장하고 사라지게 한다.
4. `Fine`, `D.C.`, `D.S.`, `Coda` 등 이동 효과가 큰 기호는 더 강한 반짝임을 준다.
5. 학습 카드의 텍스트는 교재처럼 길게 늘어놓지 말고 짧고 명확하게 배치한다.

완료 기준:

- 이벤트가 발생했을 때 플레이어가 "무슨 일이 일어났는지" 바로 안다.
- 학습 정보가 게임 분위기를 해치지 않는다.

### 8단계: 애니메이션 시스템 정리

대상 파일:

- `src/styles/animations.css`
- 필요한 경우 `src/styles/globals.css`, `src/styles/board.css`

추가할 애니메이션 예시:

```css
@keyframes tile-pop {
  0% { transform: translateY(0) scale(1); }
  45% { transform: translateY(-4px) scale(1.035); }
  100% { transform: translateY(0) scale(1); }
}

@keyframes token-bounce {
  0%, 100% { transform: translateY(0); }
  50% { transform: translateY(-7px); }
}

@keyframes hud-pulse {
  0%, 100% { box-shadow: 0 0 0 rgba(61, 139, 255, 0); }
  50% { box-shadow: 0 0 24px rgba(61, 139, 255, 0.34); }
}
```

작업:

1. 현재 턴, 현재 칸, 이벤트 발생, 주사위 굴림에만 애니메이션을 사용한다.
2. 모든 것이 동시에 움직이는 산만한 화면을 만들지 않는다.
3. 모바일에서도 성능 부담이 작도록 transform과 opacity 중심으로 구현한다.
4. `prefers-reduced-motion`를 고려해서 애니메이션을 줄이는 CSS를 추가한다.

완료 기준:

- 애니메이션이 게임 흐름을 돕는다.
- 시각 효과가 과해서 학습 내용이 묻히지 않는다.

### 9단계: 모바일 반응형

대상 파일:

- `src/styles/globals.css`
- `src/styles/board.css`

작업:

1. 1080px 이하에서는 HUD가 보드 아래로 내려가거나 sticky 하단 패널처럼 자연스럽게 배치되게 한다.
2. 640px 이하에서는 다음을 보장한다.
   - 보드가 화면 너비를 넘지 않는다.
   - 칸 안 텍스트가 겹치지 않는다.
   - 주사위 버튼은 엄지로 누르기 쉽다.
   - 이벤트 로그는 너무 길게 화면을 밀어내지 않는다.
3. 모바일 보드 칸은 기호 이미지와 한글명 중 중요한 정보가 남도록 크기를 조정한다.
4. 지나치게 작은 텍스트가 생기면 줄바꿈 또는 축약 스타일을 적용한다.

완료 기준:

- 390px 모바일 폭에서 주요 조작이 가능하다.
- 보드와 HUD가 서로 겹치지 않는다.

### 10단계: 접근성과 사용성

작업:

1. 버튼의 accessible name을 유지한다.
2. 현재 턴과 주사위 결과는 `aria-live`로 읽힐 수 있게 한다.
3. 색상만으로 상태를 구분하지 말고, 테두리/아이콘/텍스트도 함께 사용한다.
4. 포커스 링은 게임 스타일에 맞게 선명하게 보이게 한다.
5. 클릭 가능한 요소의 최소 높이는 40px 이상을 목표로 한다.

완료 기준:

- 키보드 포커스가 보인다.
- 색각 차이가 있어도 현재 상태를 파악할 수 있다.
