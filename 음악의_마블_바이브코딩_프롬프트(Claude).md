## 0. 한 줄 요약 (AI에게 주는 미션)

> 부루마블 형식의 **싱글/멀티(1~4인) 보드게임 웹앱**을 만들어라. 플레이어는 주사위를 굴려 보드를 한 칸씩 전진하고, 음악 **반복기호 칸**에 도착하면 그 기호의 실제 음악적 의미대로 말이 점프·후진·정지하며, 각 이벤트마다 **화려한 CSS 애니메이션**과 **귀여운 말 이동 연출**이 발생한다. 도착한 기호의 **음악적 의미를 설명하는 팝업**이 떠서 플레이하며 자연스럽게 음악기호를 학습한다.

학습 목표: 반복기호(도돌이표, 다카포, 달세뇨, 세뇨, 코다, 페르마타, 볼타, 다중쉼표, 8vb 등)의 **이름·기호·기능**을 게임으로 체득.

## 2. 게임 보드 데이터 모델 (★가장 중요)

보드는 **둘레를 따라 한 방향으로 도는 루프**다. 진행 방향(전진)은 인덱스 증가 방향이며,
**Start(좌상단) → 윗변(좌→우) → 오른변(위→아래) → 아랫변(우→좌) → 왼변(아래→위) → Start** 순서(시계 방향)다.

아래 `BOARD` 배열을 **게임의 정본 데이터**로 사용하라. (라벨/위치는 첨부 보드 이미지를 참고했고, 필요시 숫자만 조정 가능)

```js
// type별 의미는 4장 규칙표 참고. side는 화면 배치용.
const BOARD = [
  // ── 좌상단 코너 ──
  { id: 0,  side:'corner', type:'start',          label:'Start!',  symbol:'start' },

  // ── 윗변 (좌 → 우) ──
  { id: 1,  side:'top', type:'normal',            label:'1',       symbol:null },
  { id: 2,  side:'top', type:'normal',            label:'1',       symbol:null },
  { id: 3,  side:'top', type:'multirest',         label:'3',       symbol:'multirest', value:3 },
  { id: 4,  side:'top', type:'daCapo',            label:'D.C',     symbol:'dc' },        // → 0번(Start)로
  { id: 5,  side:'top', type:'normal',            label:'1',       symbol:null },
  { id: 6,  side:'top', type:'multirest',         label:'2',       symbol:'multirest', value:2 },
  { id: 7,  side:'top', type:'normal',            label:'1',       symbol:null },
  { id: 8,  side:'top', type:'repeatStart',       label:'||:',     symbol:'repeatStart' }, // 도돌이표 끝의 목적지
  { id: 9,  side:'top', type:'normal',            label:'1',       symbol:null },
  { id: 10, side:'top', type:'normal',            label:'1',       symbol:null },

  // ── 우상단 코너 ──
  { id: 11, side:'corner', type:'normal',         label:'',        symbol:null },

  // ── 오른변 (위 → 아래) ──
  { id: 12, side:'right', type:'volta1',          label:'1.',      symbol:'volta1' },   // → 제2괄호로
  { id: 13, side:'right', type:'repeatEnd',       label:':||',     symbol:'repeatEnd' },// → ||: 로 (뒤로)
  { id: 14, side:'right', type:'volta2',          label:'2.',      symbol:'volta2' },   // 제1괄호의 목적지
  { id: 15, side:'right', type:'segno',           label:'%',       symbol:'segno' },    // 달세뇨의 목적지
  { id: 16, side:'right', type:'coda',            label:'⊕',       symbol:'coda' },     // 첫 코다 → 둘째 코다로

  // ── 우하단 코너 ──
  { id: 17, side:'corner', type:'normal',         label:'',        symbol:null },

  // ── 아랫변 (우 → 좌) ──
  { id: 18, side:'bottom', type:'normal',         label:'1',       symbol:null },
  { id: 19, side:'bottom', type:'dalSegno',       label:'D.S',     symbol:'ds' },        // → 세뇨(%)로
  { id: 20, side:'bottom', type:'codaDest',       label:'⊕',       symbol:'coda' },      // 둘째 코다(목적지)
  { id: 21, side:'bottom', type:'octaveDown',     label:'8vb',     symbol:'8vb' },       // 옥타브 아래(뒤로)
  { id: 22, side:'bottom', type:'normal',         label:'1',       symbol:null },
  { id: 23, side:'bottom', type:'doubleSegnoTrig',label:'D.S.S',   symbol:'dss' },       // → 더블세뇨로
  { id: 24, side:'bottom', type:'normal',         label:'1',       symbol:null },
  { id: 25, side:'bottom', type:'doubleCoda',     label:'⊕⊕',      symbol:'doublecoda'}, // 첫 더블코다 → 둘째로
  { id: 26, side:'bottom', type:'doubleSegno',    label:'%%',      symbol:'doublesegno'},// 첫 더블세뇨 → 둘째로
  { id: 27, side:'bottom', type:'normal',         label:'1',       symbol:null },

  // ── 좌하단 코너 (열쇠) ──
  { id: 28, side:'corner', type:'normal',         label:'🔑',      symbol:'key' },

  // ── 왼변 (아래 → 위) ──
  { id: 29, side:'left', type:'fermata',          label:'⌒•',      symbol:'fermata' },   // 2턴 쉬기
  { id: 30, side:'left', type:'doubleSegnoDest',  label:'%%',      symbol:'doublesegno'},// 둘째 더블세뇨(목적지)
  { id: 31, side:'left', type:'doubleCodaDest',   label:'⊕⊕',      symbol:'doublecoda'}, // 둘째 더블코다(목적지)
  { id: 32, side:'left', type:'fine',             label:'Fine',    symbol:'fine' },      // 곡의 끝(축하 칸)
];
```

### 점프 목적지 해석 규칙 (코드로 동적 탐색)
하드코딩한 인덱스 대신 **type으로 칸을 찾아** 목적지를 결정하라(보드를 수정해도 안 깨지게):

```js
const findFirstByType = (t) => BOARD.find(s => s.type === t);
const findNextByType  = (t, fromId) => BOARD.find(s => s.type === t && s.id > fromId) || findFirstByType(t);

// 예) 첫 코다(coda) 도착 → 둘째 코다(codaDest)로 이동
// 예) D.S 도착 → segno 칸으로 이동
```

> 핵심 원칙: **"트리거 칸"** 은 효과가 있고, **"목적지 칸"**(`*Dest`, `segno`, `volta2`, `repeatStart`)은 효과가 없다(무한루프 방지).

## 3. 화면 레이아웃 (보드 렌더링)

부루마블식 **정사각형/직사각 둘레 보드**. CSS Grid로 둘레 셀을 배치하고 가운데는 큰 타이틀 영역.

요구사항:
- CSS Grid 14열 × 9행 정도. 둘레(테두리)에만 셀, 가운데는 `grid-column/row span`으로 비움.
- 가운데 영역에 큰 타이틀 **「반복기호의 마블」** (멀티컬러 텍스트: 글자마다 다른 색 — 초록/하늘/빨강/주황/노랑), 그 아래 주사위·턴 정보 패널.
- 각 변(side)별 테두리 색상: top=초록, right=파랑, bottom=빨강/주황, left=노랑. (첨부 이미지 색감 반영)
- 코너 칸은 더 크게(2×2 또는 강조 배경).
- 각 셀: 위쪽에 작은 라벨(다중쉼표 숫자 등), 가운데에 **기호 그래픽**, 아래에 짧은 이름.
- 반응형: 데스크톱은 둘레 보드, 모바일(좁은 화면)은 셀 크기 축소 + 가로 스크롤 또는 보드 전체 `scale()` 축소.

### 기호 렌더링 방법
픽토그램 기호는 **인라인 SVG** 권장(폰트 깨짐 방지). 텍스트 기호는 그냥 텍스트.
- SVG로 그릴 것: `repeatStart(||:)`, `repeatEnd(:||)`, `segno(%)`, `coda(⊕)`, `fermata(⌒•)`, `doublecoda(⊕⊕)`, `doublesegno(%%)`
- 텍스트로 둘 것: `D.C`, `D.S`, `D.S.S`, `Fine`, `8vb`, `1.`, `2.`, 다중쉼표 숫자
- (대안) 프로젝트에 동봉된 기호 PNG(예: `Coda.png`, `Segno.png`, `Fermata.png`, `RepeatSign_1/2.png`, `옥타브_다운…png`)를 `assets/`로 옮겨 `<img>`로 써도 된다.

**완료 기준 ①**: 보드가 둘레를 따라 그려지고, 모든 특수 칸의 기호/라벨이 보이며, 가운데 타이틀과 우측 정보 패널이 표시된다. 아직 말·주사위 동작은 없어도 된다.

## 4. 게임 규칙표 (각 칸의 효과 — 정확히 구현)

| type | 보드 표기 | 음악적 의미(설명 팝업용) | 게임 효과 |
|---|---|---|---|
| `start` | Start! | 출발 지점 | 시작 칸. 다카포 도착 시 여기로. |
| `normal` | 1 | (일반 마디) | 효과 없음. |
| `multirest` | 숫자 N | **다중쉼표**: 여러 마디를 한 번에 쉼(숫자=마디 수) | **N칸 추가 전진** (예: 4 → 4칸 앞으로) |
| `repeatStart` | ‖: | **시작 도돌이표**: 반복 구간의 시작 | 효과 없음(도돌이표 끝의 **목적지**) |
| `repeatEnd` | :‖ | **도돌이표(끝)**: 시작 도돌이표로 돌아가 반복 | **`repeatStart`(‖:) 칸으로 이동**(뒤로) |
| `volta1` | 1. | **제1괄호**: 첫 반복 때만 연주 | **`volta2`(2.) 칸으로 이동** |
| `volta2` | 2. | **제2괄호**: 반복 후 1번 건너뛰고 연주 | 효과 없음(제1괄호의 **목적지**) |
| `daCapo` | D.C | **다카포(Da Capo)**: 처음으로 돌아가 다시 연주 | **0번 Start 칸으로 이동** |
| `segno` | 𝄋 (%) | **세뇨(Segno)**: 달세뇨가 돌아올 지점 | 효과 없음(달세뇨의 **목적지**) |
| `dalSegno` | D.S | **달세뇨(Dal Segno)**: 세뇨로 돌아가 연주 | **`segno`(%) 칸으로 이동** |
| `coda` | 𝄌 (⊕) | **코다(Coda)**: 코다 기호로 건너뛰어 마지막 부분 연주 | **두 번째 코다(`codaDest`) 칸으로 이동** |
| `codaDest` | ⊕ | (코다 도착 지점) | 효과 없음 |
| `fermata` | 𝄐 (⌒•) | **페르마타(Fermata)**: 음을 자유롭게 길게 늘여 연주 | **2턴 쉬기**(다음 2번 자기 차례 건너뜀) |
| `doubleSegno` | %% | (게임 확장 기호) 더블세뇨 | **두 번째 더블세뇨(`doubleSegnoDest`)로 이동** |
| `doubleSegnoDest` | %% | (도착 지점) | 효과 없음 |
| `doubleSegnoTrig` | D.S.S | (게임 확장) 더블세뇨로 보내는 트리거 | **`doubleSegno` 칸으로 이동** |
| `doubleCoda` | ⊕⊕ | (게임 확장 기호) 더블코다 | **두 번째 더블코다(`doubleCodaDest`)로 이동** |
| `doubleCodaDest` | ⊕⊕ | (도착 지점) | 효과 없음 |
| `octaveDown` | 8vb | **옥타브 아래(Ottava bassa)**: 한 옥타브 낮게 | **8칸 뒤로 이동**(0번 미만이면 0번에서 멈춤) |
| `fine` | Fine | **피네(Fine)**: 곡의 끝 | 축하 연출 + (선택)완주 보너스. 효과는 정보성. |

### 턴 진행 흐름 (state machine)
```
idle → (주사위 굴림) rolling → moving(말이 한 칸씩 hop) 
     → landed(도착 칸 효과 판정)
        ├─ 효과 없음 → 설명팝업(있으면) → nextTurn
        └─ 점프/후진/쉬기 → 트리거 애니메이션 → 말 이동 → 도착 칸 재판정*
     → nextTurn
```
\* **재판정 루프 방지**: 한 턴에 트리거가 연쇄 발동할 수 있으나, 목적지 칸(`*Dest`/segno/volta2/repeatStart)은 효과가 없으므로 자연히 멈춘다. 안전장치로 **한 턴 최대 연쇄 5회** 가드를 둬라.

### 조작
- **주사위 굴리기**: 화면 버튼 클릭 또는 `Space` 키 → 1~6 무작위.
- **수동 1칸 전진**: `→`(오른쪽 방향키) → 한 칸만 앞으로(테스트/연습용). 후진은 불가(앞으로만).
- **멀티플레이(1~4인)**: 턴제(핫시트). 현재 플레이어 강조 표시. 인원 수 시작 화면에서 선택.

### 승리 조건(조정 가능, 기본값 제시)
- 기본: **보드를 한 바퀴 돌아 Start(0번)를 다시 통과/도착**하면 그 플레이어 "완주!" → 게임 종료 & 우승.
- 페르마타로 쉬는 동안 다른 플레이어가 먼저 완주할 수 있음.
> 이 조건은 명확히 분리된 함수(`checkWin`)로 만들어, 추후 "2바퀴" 등으로 쉽게 바꿀 수 있게 하라.

**완료 기준 ②**: 주사위/방향키로 말이 움직이고, 위 표의 **모든 효과가 정확히** 동작한다(점프·후진·2턴 쉬기·완주 판정). 애니메이션은 단순해도 좋다.

## 5. 애니메이션 명세 (★몰입 포인트)

### 5-1. 말 이동 — "귀여운 통통 이동" (요청하신 CGV 느낌 = 캐릭터가 통통 튀는 이동 연출로 해석)
- 한 칸 이동마다 **포물선 hop**: 위로 살짝 떠올랐다가(`translateY` -18px) 다음 칸에 착지.
- **스쿼시 & 스트레치**: 도약 직전 살짝 납작(`scaleY .8`), 공중에서 길쭉(`scaleY 1.1`), 착지 시 다시 납작 후 복원.
- 칸당 약 **260~340ms**, `cubic-bezier(.34,1.56,.64,1)`(살짝 튕기는 이징).
- 착지마다 작은 **먼지/별 파티클** + 미세한 화면 흔들림(선택).
- 말 디자인: 음표(♪) 모양 + 눈/표정이 있는 귀여운 캐릭터(SVG). 이동 중 눈 깜빡임/방향 기울기.
- 효과음(선택): 착지마다 짧은 "뽁" 사운드.

```css
@keyframes hop {
  0%   { transform: translate(0,0) scaleY(.85); }
  45%  { transform: translate(var(--dx), -18px) scaleY(1.12); }
  100% { transform: translate(var(--dx), 0) scaleY(1); }
}
```
> 구현 팁: 각 칸의 화면 좌표를 미리 계산해두고(`getBoundingClientRect`), 말을 절대좌표로 두 칸 사이를 순차 이동. 여러 칸 이동 시 `for…await` 로 한 칸씩 hop 재생.

### 5-2. 주사위
- 클릭 시 **3D 큐브 텀블**(또는 빠른 숫자 페이스 교체) ~700ms 후 결과에서 정지.
- 결과 숫자 큰 **팝 인** + 글로우.

### 5-3. 기호 칸 도착 이벤트 — 기호별 테마 애니메이션
각 효과 발동 전에 **풀스크린 오버레이 연출**(0.8~1.5초) 후 말 이동을 실행하라. 모두 화려한 CSS로.

| 기호 | 연출 컨셉 |
|---|---|
| **도돌이표(:‖)** | "되감기 ⏪": 화면에 ⏪ 아이콘, 말이 잔상(모션 블러)을 남기며 **빠르게 거꾸로** ‖: 칸으로 슉. 청록 스트라이프 와이프. |
| **다카포(D.C)** | "처음으로!": 카메라가 보드를 쭉 패닝하듯 보드가 살짝 줌아웃→Start로 시선 이동, Start에 도착 시 **색종이 폭죽(confetti)**. "D.C" 대형 텍스트 슬라이드. |
| **달세뇨(D.S)/세뇨(%)** | "소용돌이 🌀": 세뇨 기호가 회전하며 커지고, 말이 **나선(spiral)** 궤적으로 세뇨 칸까지 빨려 들어감. 보라/남색 스월. |
| **코다(⊕)/더블코다(⊕⊕)** | "워프 점프 ✨": 화면 중앙에서 원형 **아이리스(iris) 줌**, 코다 기호 펄스 + 파티클 버스트, 말이 순간이동하듯 도약. 더블코다는 효과 2배(이중 링 + 파티클 더 많이). |
| **제1괄호(1.) → 제2괄호(2.)** | "스킵 ⏭": 1번 괄호 박스가 X로 지워지고, 말이 **점선 점프 아크**로 2번 괄호로 슬라이드. |
| **페르마타(⌒•)** | "정지/홀드 ⏸": 말 위로 **페르마타 후광**이 생기고 슬로우 펄스, "쉬는 중 2턴" 모래시계/배지 표시, 주변이 살짝 어두워졌다 밝아짐. |
| **다중쉼표(N)** | "대시 💨": 숫자 N이 크게 팝, 말이 **N칸을 빠른 연속 hop**으로 질주(잔상). |
| **8vb (옥타브 아래)** | "드롭 🔽": 화면 톤이 한 단계 어두워지고 음정이 "뚝" 떨어지는 느낌으로 말이 **아래로 떨어지듯** 뒤로 이동. 파란 하강 화살표. |
| **Fine** | "피날레 🎉": 황금빛 샤이머 + 큰 폭죽 + "완성!" 텍스트. (완주면 우승 화면) |
| **더블세뇨(%%)/D.S.S** | "더블 소용돌이": 5-3 달세뇨 스월을 **이중 회전**으로 강화. |

> 라이브러리: 폭죽은 `canvas-confetti`. 나머지는 CSS `@keyframes` + Web Animations API(`element.animate()`)로 충분. (React면 Framer Motion `variants` 사용)

### 5-4. 기호 설명 팝업(학습 핵심)
이벤트 연출이 끝나면 **모달/토스트**로 해당 기호 카드를 띄운다:
- 큰 기호 그래픽 + 이름(한글/영어) + 4장 규칙표의 "음악적 의미" 한두 문장.
- "확인" 누르면 닫히고 다음 턴. (자동 3초 후 닫힘 옵션)
- 같은 기호를 다시 밟으면 팝업은 더 짧게(또는 토스트만).

**완료 기준 ③**: 모든 기호 이벤트에 5장 컨셉의 전용 애니메이션과 설명 팝업이 연결되어, 도착할 때마다 시각적으로 다르게 연출된다.

## 7. 접근성 & 반응형
- 키보드: `Space`=주사위, `→`=수동 1칸, `Enter`=팝업 닫기. 포커스 링 유지.
- **ARIA live region**(`aria-live="polite"`)으로 주사위 결과·이벤트를 음성 안내.
- 색만으로 구분 금지: 기호엔 항상 텍스트 라벨 동반.
- 모바일: 버튼은 큰 탭 영역, 보드는 `transform: scale()`로 화면에 맞게 축소.
- `prefers-reduced-motion` 사용자에겐 hop/연출을 짧은 페이드로 대체.

## 8. 구현 순서 요약 (Phase 체크리스트)

- [ ] **Phase 1 — 보드 렌더링**: `BOARD` 데이터로 둘레 그리드 + 기호/타이틀/패널. (완료기준 ①)
- [ ] **Phase 2 — 말 & 이동**: 말 1개 + `→` 한 칸 전진 + hop 애니메이션. 칸 좌표 계산.
- [ ] **Phase 3 — 주사위**: 굴림 애니메이션 + 결과만큼 연속 hop 이동.
- [ ] **Phase 4 — 기호 효과 로직**: 4장 규칙표 전부 구현(점프/후진/2턴쉬기/연쇄가드/완주). (완료기준 ②)
- [ ] **Phase 5 — 이벤트 애니메이션 + 설명 팝업**: 5장 기호별 연출 연결. (완료기준 ③)
- [ ] **Phase 6 — 멀티플레이/마무리**: 1~4인 턴제, 승리 화면, 소리, 접근성, 반응형, 다시시작.

> 각 Phase 끝에 직접 실행해 동작을 확인하고, 통과 후 다음 단계 코드를 추가하라. 한 번에 전체를 쏟아내지 말 것.

## 9. 테스트 시나리오 (꼭 통과)
1. Start에서 주사위 4 → 4번(D.C) 도착 → 즉시 0번(Start)로 복귀 + 폭죽.
2. 13번(:‖) 도착 → 8번(‖:)으로 **뒤로** 이동(되감기 연출).
3. 12번(1.) 도착 → 14번(2.)으로 이동.
4. 16번(⊕) 도착 → 20번(⊕)으로 이동(워프).
5. 19번(D.S) 도착 → 15번(%)으로 이동(스월).
6. 29번(페르마타) 도착 → 다음 **2번** 자기 차례를 건너뜀.
7. 21번(8vb) 도착 → 8칸 뒤로(13번), 그 13번(:‖) 효과가 **연쇄** 발동해 8번으로(연쇄 가드 정상 동작 확인).
8. 3번(다중쉼표 3) 도착 → 추가로 3칸 전진.
9. 보드 한 바퀴 완주 시 우승 화면.
10. `prefers-reduced-motion`에서 연출이 깨지지 않고 짧게 처리됨.
