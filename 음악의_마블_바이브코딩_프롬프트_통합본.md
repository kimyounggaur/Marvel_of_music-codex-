# 🎵🎲 「반복기호의 마블」 — 음악이론 보드게임 웹앱 · 바이브코딩 마스터 프롬프트 (통합본)

> **목적**: 부루마블 형식의 보드게임을 즐기며 음악 **반복·이동 기호**(도돌이표·다카포·달세뇨·세뇨·코다·페르마타·볼타·다중쉼표·8vb 등)를 자연스럽게 익히는 **교육용 웹앱**을 만든다.
> **사용법**: 이 문서를 Cursor / Claude Code / v0 / Lovable / Replit Agent / ChatGPT 등에 **0장(마스터 프롬프트)부터 단계별로** 붙여 넣어 개발한다. 한 Phase가 통과되면 다음으로 넘어간다.
>
> **설계 철학(3대 원칙)**
> 1. **규칙은 데이터로** — 기호 효과를 `if`문 난립이 아니라 `BOARD` 데이터 + 분리된 규칙 엔진으로 관리(테스트 가능).
> 2. **이동은 한 칸씩, 보이게** — 순간이동 금지. 주사위 → 방향키로 한 칸씩 hop → 도착 후 이벤트.
> 3. **학습은 연출로** — 모든 이벤트에 ① 기호 의미에 맞는 **화려한 애니메이션** ② **학습 카드(실제 음악적 의미 + 기억법)** 를 붙인다.

## 0. 🧠 마스터 프롬프트 (AI 에이전트에게 한 번에 주는 역할/요구사항)

```text
너는 어린이·청소년 음악교육용 웹게임을 만드는 시니어 프론트엔드 개발자이자 인터랙티브 애니메이션 전문가, UI/UX 디자이너다.

프로젝트명은 "음악의 마블(반복기호의 마블)"이다. 부루마블처럼 사각 보드 위를 한 방향으로 전진하며 음악 반복기호를 배우는 웹앱을 만든다.

[핵심 요구사항]
1. 보드는 사각형 테두리형 보드판이며, 말은 한 방향으로만 전진한다.
2. 주사위 굴리기 버튼으로 1~6을 뽑는다.
3. 주사위 결과(N)가 나오면 "남은 이동 수 = N"이 표시되고, 사용자가 우측 방향키(ArrowRight)
   또는 화면의 "한 칸 이동" 버튼을 누를 때마다 정확히 한 칸씩 이동한다(자동 순간이동 금지).
4. 남은 이동 수가 0이 되는 순간, 도착한 칸의 음악기호 이벤트를 실행한다.
5. 말이 한 칸 이동할 때마다 귀여운 통통 튀는(hop) SVG/CSS 애니메이션이 적용된다.
6. 음악기호 칸 도착 시, 기호의 실제 의미에 부합하는 "화려한" CSS 애니메이션 연출이 발생한다.
7. 이벤트가 끝나면 학습 카드(또는 토스트)로 "실제 악보에서의 의미 + 게임 효과 + 한 줄 기억법"을 보여준다.
8. 모바일에서도 쓰도록 큰 "한 칸 이동" 터치 버튼을 제공한다.
9. 게임 규칙은 boardCells 데이터 + 순수 함수 규칙 엔진으로 분리해 UI와 독립적으로 단위 테스트가 가능하게 한다.
10. 이미지 에셋이 없거나 로딩 실패하면 텍스트/유니코드 fallback으로 표시한다.
11. 접근성(버튼 aria-label, 키보드 조작, prefers-reduced-motion)과 반응형을 고려한다.
12. 이벤트 이동(점프/후진/추가전진) 후 도착 칸에 또 이벤트가 있으면 이어서 처리하되,
    무한 루프 방지를 위해 chainDepth를 최대 10으로 제한하고, 목적지 전용 칸은 markerOnly=true로 둬서 효과가 다시 발동하지 않게 한다.
13. 이동 중/주사위 굴리는 중/이벤트 연출 중/페르마타 쉬는 중에는 입력 버튼을 비활성화한다.

[디자인 톤]
- 노션처럼 깔끔하고 미니멀한 모바일 친화 UI를 베이스로 하되, 보드판과 이벤트 연출은 화려하고 직관적으로.
- 말은 귀여운 음표/음악요정 캐릭터(SVG). 색상은 밝고 명랑하게, 초등학생도 쓰기 쉬운 큰 버튼·큰 글씨.

[기술 스택 - 권장]
- React + TypeScript + Vite (상태관리는 useReducer). 외부 UI 라이브러리 의존 최소화.
- 애니메이션은 순수 CSS @keyframes + Web Animations API. 폭죽만 canvas-confetti(CDN) 허용.
- 규칙 엔진 테스트는 Vitest.
(※ 빌드 없이 단일 index.html로 만들고 싶으면 1장 "단일 HTML 대안"을 따른다. 데이터/규칙/애니메이션 명세는 두 방식 모두 동일하게 재사용한다.)

[구현 순서]
한 번에 전부 쏟지 말고, 이 문서의 Phase 1~6 순서대로 만들고 각 Phase 완료 기준을 보고한 뒤 다음으로 진행하라.
```

## 2. 🎨 디자인 시스템

- **분위기**: 음악 교실 + 보드게임 + 귀여운 아케이드. 밝고 명랑.
- **레이아웃**: 데스크톱은 보드 좌측 + 조작 패널 우측, 모바일은 보드 위 + 조작 아래(sticky).

## 3. 🗺️ 정본 보드 데이터 & 타입 (★가장 중요)

원본 보드(김영걸 A2)를 반영한 **40칸 사각 보드**. 진행 방향은 인덱스 증가(시계 방향):
**Start(좌상단) → 윗변(좌→우) → 오른변(위→아래) → 아랫변(우→좌) → 왼변(아래→위) → Start.**

### 3-1. 타입 (`src/game/types.ts`)
```ts
export type CellKind =
  | 'START' | 'NORMAL' | 'MULTI_REST'
  | 'REPEAT_START' | 'REPEAT_END'
  | 'DAL_SEGNO' | 'SEGNO'
  | 'CODA' | 'DA_CAPO'
  | 'FIRST_ENDING' | 'SECOND_ENDING'
  | 'FERMATA'
  | 'DOUBLE_SEGNO' | 'DOUBLE_SEGNO_TRIGGER'
  | 'DOUBLE_CODA'
  | 'OCTAVE_DOWN' | 'FINE';

export interface BoardCell {
  index: number;
  kind: CellKind;
  label: string;            // 칸에 표기할 짧은 텍스트/기호
  side: 'top'|'right'|'bottom'|'left';
  isCorner?: boolean;
  markerOnly?: boolean;     // 목적지 전용(효과 없음) → 무한루프 방지
  count?: number;           // MULTI_REST 전진 칸 수
  pairId?: string;          // 짝 매칭 (coda/dcoda/dsegno/rep)
  occurrence?: 1 | 2;       // 첫 번째/두 번째
}

export type GamePhase =
  | 'READY' | 'ROLLING' | 'WAITING_STEP' | 'MOVING'
  | 'RESOLVING_EVENT' | 'SKIPPING' | 'GAME_OVER';

export type ResolvedEventType =
  | 'NONE' | 'MOVE_FORWARD' | 'MOVE_BACKWARD'
  | 'JUMP_TO_INDEX' | 'SKIP_TURNS' | 'GAME_FINISH';

export interface ResolvedEvent {
  type: ResolvedEventType;
  sourceIndex: number;
  targetIndex?: number;
  steps?: number;
  skipTurns?: number;
  animationClass: string;   // 8장 fx-* 클래스
  kind: CellKind;
}

export interface Player { id:number; name:string; color:string; position:number; skipTurns:number; finished:boolean; }

export interface GameState {
  players: Player[];
  currentPlayer: number;
  diceValue: number | null;
  remainingSteps: number;
  phase: GamePhase;
  activeEvent: ResolvedEvent | null;
  eventLog: string[];
  chainDepth: number;
  turnCount: number;
}
```

### 3-2. 좌표 계산 (`src/game/movement.ts`) — 14열 × 8행 사각 둘레
```ts
export const COLS = 14, ROWS = 8;            // 둘레 = 2*(14+8)-4 = 40칸
export function cellCoord(i:number){          // index → {row,col} (0-based)
  if (i <= 13) return { row:0, col:i };                 // 윗변 (좌→우)
  if (i <= 19) return { row:i-13, col:13 };             // 오른변 (위→아래)
  if (i <= 33) return { row:7, col:13-(i-20) };         // 아랫변 (우→좌)
  return { row:6-(i-34), col:0 };                       // 왼변 (아래→위)
}
export const stepForward = (pos:number, len:number)=>(pos+1)%len;        // 앞으로 1칸
export const stepBackward = (pos:number, n:number, len:number)=>((pos-n)%len+len)%len;
```
> ArrowRight는 "물리적 오른쪽"이 아니라 **경로상 다음 칸**으로 이동이다. 중앙 영역(row 1~6, col 1~12)에는 타이틀/안내/이벤트 요약을 배치한다.

## 4. ⚙️ 규칙 엔진 (`src/game/rules.ts`) — 순수 함수

```text
resolveCellEvent(board, position): ResolvedEvent 를 구현하라.

공통: markerOnly === true 이거나 kind가 NORMAL/START/SEGNO/SECOND_ENDING/REPEAT_START 면 type:'NONE'.

각 kind별 반환:
- MULTI_REST          → MOVE_FORWARD, steps:cell.count,        animationClass:'fx-rocket-dash'
- REPEAT_END          → JUMP_TO_INDEX, target=같은 pairId의 REPEAT_START index, 'fx-rewind'
- DAL_SEGNO           → JUMP_TO_INDEX, target=SEGNO index,      'fx-segno-fly'
- CODA(occurrence:1)  → JUMP_TO_INDEX, target=같은 pairId의 occurrence:2 index, 'fx-coda-portal'
- DA_CAPO             → JUMP_TO_INDEX, target=0,                'fx-spotlight-start'
- FIRST_ENDING        → JUMP_TO_INDEX, target=SECOND_ENDING index, 'fx-spring-jump'
- FERMATA             → SKIP_TURNS, skipTurns:2,                'fx-ice-freeze'
- DOUBLE_SEGNO(occ:1) → JUMP_TO_INDEX, target=dsegno occ:2,     'fx-hologram'
- DOUBLE_SEGNO_TRIGGER→ JUMP_TO_INDEX, target=DOUBLE_SEGNO occ:1, 'fx-hologram'
- DOUBLE_CODA(occ:1)  → JUMP_TO_INDEX, target=dcoda occ:2,      'fx-wormhole'
- OCTAVE_DOWN         → MOVE_BACKWARD, steps:8,                 'fx-octave-drop'
- FINE                → GAME_FINISH,                            'fx-finale'

목적지(target)를 못 찾으면 throw 하지 말고 NONE + 경고 로그를 반환한다(앱이 죽지 않게).

보조 함수:
- findIndexByKind(board, kind)
- findPair(board, cell)  // pairId+occurrence로 짝 찾기
- getPositionAfterSteps(pos, steps, len)  // 앞/뒤 공통
```

**핵심 안전장치**
- `markerOnly` 칸은 항상 NONE → 점프 목적지에서 이벤트 재발동 없음.
- 이벤트 이동 후 도착 칸을 **재판정**(연쇄 가능)하되 `chainDepth ≤ 10` 가드.
- 예시 연쇄(정상 동작이어야 함): `8vb(23) → 8칸 뒤로 → 15(:‖) → 8(‖:) marker 정지` / `다중쉼표(3)@3 → 6(다중쉼표2) → 8(‖:) 정지` / `다중쉼표(4)@30 → 34(페르마타) → 2번 쉬기`.

## 5. 🔁 상태 머신 & reducer (`src/game/reducer.ts`)

`useReducer` 기반 **순수 함수**. `setTimeout` 등 부수효과는 `hooks/useGameLoop.ts`에서 처리(반드시 cleanup).

**액션**: `ROLL_DICE_START` · `ROLL_DICE_COMPLETE(value)` · `STEP_FORWARD_START` · `STEP_FORWARD_COMPLETE` · `RESOLVE_EVENT_START` · `APPLY_EVENT_MOVE(target|steps)` · `APPLY_SKIP_TURNS(n)` · `EVENT_COMPLETE` · `NEXT_TURN` · `RESET_GAME` · `FINISH_GAME` · `ADD_LOG`.

**흐름**
```
READY
 └ 주사위 클릭/Space
     ├ 현재 플레이어 skipTurns>0 → skipTurns-- , 로그("페르마타로 쉬었어요. 남은 쉬기 n") → NEXT_TURN
     └ skipTurns==0 → ROLLING (0.7s) → diceValue 확정 → remainingSteps=value → WAITING_STEP
WAITING_STEP
 └ ArrowRight/한 칸 이동 버튼 → STEP_FORWARD_START → (hop 0.3s) → STEP_FORWARD_COMPLETE
     ├ remainingSteps>0 → 다시 WAITING_STEP
     └ remainingSteps==0 → RESOLVING_EVENT
RESOLVING_EVENT
 └ resolveCellEvent()
     ├ NONE        → 학습카드(있으면) → NEXT_TURN
     ├ MOVE_FORWARD/BACKWARD → fx 연출 후 steps만큼 빠른 연속 이동(칸당 0.18~0.25s) → 도착 후 재판정
     ├ JUMP_TO_INDEX → fx 연출(0.8s) → target으로 이동 → 도착 후 재판정
     ├ SKIP_TURNS  → 현재 플레이어 skipTurns += 2 → 학습카드 → NEXT_TURN
     └ GAME_FINISH → GAME_OVER
 └ chainDepth>10 → 연쇄 중단, NEXT_TURN
```
- 입력 잠금: `ROLLING/MOVING/RESOLVING_EVENT/SKIPPING/GAME_OVER`에서는 주사위·이동 버튼 비활성화.
- 로그는 최신 20개만 유지.

## 6. 🎲 주사위 & 조작 (`DicePanel.tsx`, `dice.ts`)

1. "주사위 굴리기" 버튼 + 모바일용 큰 "한 칸 이동" 버튼.
2. 굴리는 동안 0.7초간 1~6 눈이 빠르게 바뀌는 **3D 큐브 텀블** 애니메이션 → 결과에서 정지, 결과 숫자 팝인+글로우.
3. "남은 이동 수"를 크게 표시. `remainingSteps>0`이면 "→ 또는 버튼으로 이동하세요" 안내.
4. 키보드: `ArrowRight`=한 칸 이동, `Space/Enter`=주사위(READY일 때), 팝업 `Enter`=닫기.
5. `dice.ts`의 `rollDice()`는 `Math.random` 기반이되 **함수로 분리**해 테스트 가능하게.
6. 버튼 비활성 조건은 5장 입력 잠금 규칙을 그대로 따른다.

**완료 기준 ②**: 주사위 → 방향키로 한 칸씩 이동 → 도착 칸 이벤트(점프/후진/추가전진/2턴 쉬기/완주)가 4장 규칙대로 **정확히** 동작한다. (연출은 단순해도 됨)

## 7. 🐣 말 캐릭터 & 이동 애니메이션 (`PlayerToken.tsx`)

- **캐릭터**: 귀여운 음표/음악요정(둥근 얼굴, 작은 음표 꼬리, 눈 깜빡임). 인라인 SVG 권장. 플레이어별 색.
- **한 칸 이동(hop)**: 포물선으로 살짝 떠올랐다가 착지 + 스쿼시&스트레치 + 착지 시 작은 별/먼지 파티클 + 말 아래 그림자 신축.
```css
@keyframes token-hop{
  0%{ transform:translate(0,0) scaleY(.85); }
  45%{ transform:translate(var(--dx),var(--dy-up)) scaleY(1.12); }
  100%{ transform:translate(var(--dx),var(--dy)) scaleY(1); }
}
```
- 칸당 **260~340ms**, `cubic-bezier(.34,1.56,.64,1)`. 여러 칸은 `for…await`로 한 칸씩 순차 재생.

- keyframes 이름: `token-idle-bounce`, `token-hop`, `token-land-pop`, `token-rewind-spin`, `token-portal-warp`, `token-freeze`.
- `prefers-reduced-motion`: 큰 transform 대신 짧은 fade/outline 변화로 대체.
